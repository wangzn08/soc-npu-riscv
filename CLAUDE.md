# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

SoC simulation project: a PicoRV32 RISC-V CPU and a 16×16 systolic-array NPU sharing memory over AXI, verified in ModelSim/Questa. Firmware (C + asm, rv32imc) runs on the simulated CPU and drives the NPU through MMIO registers. The end goal is DeepConvNet (MNIST) inference offloaded to the NPU.

## Build & Simulation Commands

The Makefile uses POSIX shell commands (`mkdir -p`, `chmod`, `rm -rf`) — run it from an MSYS/Git Bash environment on Windows, not PowerShell/cmd.

```
make fw         # compile firmware → firmware/build/firmware7.hex
make compile    # compile RTL into sim/work (vlog -f axi_sys.f)
make sim        # compile + run headless simulation (vsim -c, run -all)
make waves      # compile + run GUI simulation, dump VCD
make clean      # remove simulation artifacts
make distclean  # also remove firmware/build
```

Tool paths (override via environment/make variables if needed):
- `RISCV_PREFIX` — defaults to `E:/Riscv_Tools/xpack-riscv-none-elf-gcc-15.2.0-1/bin/riscv-none-elf-`
- ModelSim at `E:/modelsim` (`MGC_LICENSE_FILE` is exported by the Makefile)

There is no separate unit-test framework. "The test" is one simulation run: the testbench (`rtl/axi_sys_tb.v`) prints `ALL TESTS PASSED.` when firmware writes the magic value `123456789` to MMIO address `0x2000_0000`, or `ERROR!`/`TIMEOUT` otherwise. Firmware `print_*` output appears on the simulator console via the UART MMIO.

Important: `vsim` must run from the repo root — the firmware hex path (`firmware/build/firmware7.hex`) is resolved relative to the simulation working directory by `$readmemh`. A different hex can be passed with the `+firmware=<path>` plusarg.

`make compile` only re-runs when `sim/work` doesn't exist; after editing RTL, run `make clean` first (or `vlog` the changed file manually).

## Architecture

### System topology (rtl/axi_sys.v is the integration hub)

```
picorv32_axi (AXI-Lite master)
  ├─ 0x0000_0000–0x00FF_FFFF → axi_lite_ram (private mem, firmware loaded here via $readmemh)
  ├─ 0x1000_0000             → UART tx (write char, simulation $write)
  ├─ 0x2000_0000             → test-pass register (write 123456789 → tests_passed)
  ├─ 0x3000_0000–0x3000_0FFF → NPU registers (forwarded to npu_axi_wrapper, NOT a normal AXI slave)
  └─ 0x4000_0000–...         → shared memory ("DDR"):
        axi_lite_to_axi_full bridge → axi_arbiter_2to1 ← npu DMA master
                                          └→ axi_full_slave_v1_0_S00_AXI
```

- NPU register access is special-cased in `axi_sys.v`: MMIO writes/reads are converted into a `reg_wr_en/reg_rd_en` pulse interface into `npu_axi_wrapper` (byte address → word address via `addr[11:2]`), with handshake stretching until `reg_wr_done`/`rd_data_valid`. New NPU registers go in `rtl/param_regfile.v` and must be mirrored in the `NPU_*` defines in `firmware/firmware.h`.
- NPU `irq_done` is latched in `axi_sys.v` and fed to PicoRV32 IRQ bit 3; `firmware/irq.c` sets `npu_irq_flag` for the main loop to poll.
- `rtl/npu_axi_wrapper.v` adapts the NPU's 128-bit AXI master to the SoC's 32-bit bus.

### NPU internals (rtl/npu_top.v)

Dataflow: `axi_dma` loads activations/weights from shared memory into ping-pong SRAMs (`sram_models.v`) → `im2col_line_buffer` + `wgt_reader` feed `systolic_16x16` (built from `gp_4x4` → `pe_core`) → `post_process_top` (bias/quant/ReLU + `max_pooling_2x2`, `vector_alu` for eltwise add) → output SRAM → DMA back to shared memory. `top_controller_fsm` sequences everything; `param_regfile` holds the config/status registers.

### RTL file list

All simulated RTL is listed in `axi_sys.f` (compiled with `vlog -f`). New RTL files must be added there.

### Firmware

- Linked objects are fixed in the Makefile: `start7.S` + `irq.c` + `print.c` + `libgcc_stub.c` + `deepnet_deploy.c`. The entry point called from `start7.S` is `usercode7()`.
- Multiple alternative test programs exist (`deepnet_run.c` — DDR/DMA path tests; `deepnet_deploy.c` — current build), each defining `usercode7()`. Only one may be linked at a time; switch by editing `FW_C_SRCS` in the Makefile.
- Freestanding environment: no libc (`-nostdlib -ffreestanding`), `libgcc_stub.c` provides helpers, `print.c` provides console output. CFLAGS are strict (`-Werror -Wall -Wextra -pedantic` plus many more) — new firmware code must be warning-clean.
- Memory layout: code/data at 0x0 in private RAM (`sections.lds`), stack grows down from top (STACKADDR 0xffff_ffff wraps into private RAM). `makehex.py` pads the binary to 524288 words for `$readmemh`.
- DeepConvNet assets: `deepnet.h` (layer dimensions), `deepnet_weights.h`, `mnist_test_images.h` (generated by `extract_images.py`).

## Conventions

- Comments in RTL and firmware are bilingual (Chinese/English); keep matching the surrounding style.
- Register-map changes require touching both sides: `rtl/param_regfile.v` (and possibly `top_controller_fsm.v`) and `firmware/firmware.h`.
