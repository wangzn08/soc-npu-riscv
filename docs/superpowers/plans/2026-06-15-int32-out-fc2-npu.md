# INT32 raw output → FC2 (final classifier) on NPU — decision Q

> Branch: `feat/npu-oc-single-start`. Bit-identical gate: `SCORE_CHK == D30179DF`
> + 10/10 every step. Goal: run ALL FC on NPU (remove decision F's "FC2 stays CPU").

**Why:** FC2 (50→10) on the CPU costs ~22.3K cyc/img (~20% of inference) — 500 MACs
at ~44 cyc/MAC because PicoRV32 is a slow multicycle core. The NPU array does those
MACs in dedicated INT8 hardware. FC2 stayed on CPU only because the post-process
clamps to INT8, saturating the final logits. **Decision Q adds a raw-INT32 output
mode** (`int32_out`, CTRL[13]) so the NPU can emit the un-clamped scaled INT32 result
— a general capability (any final-classifier / un-quantized layer), then moves FC2
to the NPU.

**Key design (verified against code):**
- Capture point = **`s2_quant`** (post_process stage-2: `(psum+bias)*scale >>> shift`,
  full INT32, BEFORE the S3 INT8 clamp). This equals the CPU FC2's
  `(acc*SCALE_AFFINE2) >> SCALE_SHIFT` exactly ⇒ bit-identical. (Same 64-bit signed
  product / arithmetic-shift / low-32 path FC1 already matches its CPU oracle on.)
- Output packing: 16 OC × INT32 = 512 bits → **4 Out-SRAM words** (4×INT32 each),
  serialized by a sequencer; DMA reads 4 words back instead of 1.
- Scope: GEMM/FC (`int32_out` is firmware-set for FC2). Conv/INT8 path untouched.
- FC2 weights are ALREADY preloaded (`FC2_WGT_BASE`, pack_fc_tile, deploy code).

## Steps (each: clean build + full sim, SCORE_CHK D30179DF + 10/10, commit)
- [ ] **Q1. `int32_out` CTRL[13]** in regfile + firmware.h + npu_top wire (dormant).
  Byte-identical. Commit.
- [ ] **Q2. post_process expose `o_feat32`** (16×INT32 = s2_quant) + valid; dormant
  (npu_top ignores). Byte-identical. Commit.
- [ ] **Q3. npu_top INT32 write sequencer** — in `int32_out` mode, on the GEMM
  output-write, serialize `o_feat32` into 4 Out-SRAM words at the GEMM out base;
  off ⇒ byte-identical. Build byte-identical. Commit.
- [ ] **Q4. Firmware: FC2 on NPU** — `npu_gemm_pass(..., int32_out)`; FC2 path:
  DMA FC1 output → Act, GEMM 50→10 (scale=SCALE_AFFINE2, bias=affine2_b, relu off,
  int32_out=1) → 4 Out words → DMA to DDR → CPU reads 10 INT32 for argmax. Remove the
  CPU FC2 MAC loop. Verify SCORE_CHK == D30179DF, 10/10.
- [ ] **Q5. Measure** full-run A/B + per-image inference; document decision Q in
  CLAUDE.md + memory; note FC2 CPU→NPU delta and the general INT32-output capability.

## Risk / rollback
- Each step mode-gated (`int32_out` off ⇒ byte-identical). If Q4 can't reach
  D30179DF, revert FC2 to the CPU loop — the mode bit stays off.
- The 4-word serialized write + 4-word DMA is the main new sequencing; bring up the
  HW (Q3) byte-identical (off) before wiring FC2 (Q4).
- Bit-identical only if NPU s2_quant == CPU `(acc*SCALE)>>SHIFT` (bias=affine2_b,
  scale_mul=SCALE_AFFINE2, scale_shift=SCALE_SHIFT, IC=50, OC=10). FC1 already
  validates this arithmetic path.
